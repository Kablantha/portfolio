Project Overview

STATUS: FINAL · REVISION: POLISHED

This project is a recruitment-focused web experience for Metro State University’s Learner Experience Design Graduate Certificate. It focuses on how program information is communicated, especially to working adults who are considering a return to school. The goal was to design an experience that helps prospective students quickly understand what the certificate is, how it fits into a working adult’s life, and whether it is a realistic option for them.

Problem & Audience

CONTEXT: USER NEED · PRIMARY ISSUE: ACCESSIBILITY

The core problem this project addresses is that the existing informational webpage assumes a level of comfort with academic language, which can feel intimidating or exclusionary to working adults. While program details were present, the experience did not account for users navigating limited cognitive energy or doubts about belonging. The intended audience includes adults balancing work and personal responsibilities who need clarity, reassurance, and realistic information to make an informed decision.

User Insights

RESEARCH TYPE: INTERVIEW · IMPACT: DIRECT

User insights for this project were informed by an interview with a student who had recently navigated the decision to enroll in the Learner Experience Design Graduate Certificate. She emphasized that remote classes were essential rather than optional and expressed concerns about belonging, age, and a perceived lack of technical experience. A specific quote from the interview stood out to me: "We designed a functional prototype. I wouldn’t have known what these words even meant six months ago." This showed me how a student’s perceived technical ability can discourage them before they begin. These insights directly shaped the project’s emphasis on plain language, contextual explanations, and a reassuring tone that supports growth rather than assuming prior expertise.

Communication Approach

TONE: SUPPORTIVE · STRUCTURE: WEB EXPERIENCE

I approached the page as if it were written by a guidance counselor for adults: supportive, honest, and non-coercive. This guided my choices around tone, structure, and content, including the use of plain language, explicit labeling, and reassurance without exaggeration. Rather than marketing the program, the page prioritizes helping users make a realistic, informed decision based on their own needs and constraints.

The choice to create a web experience rather than a static document or brochure was driven by accessibility. Working adults often suffer from decreased cognitive energy, especially at the end of a long workday. A web experience gave me the flexibility to design for this reality by using progressive disclosure to limit how much information is visible at once, which allows users to engage with the content at their own pace.

Access & Inclusion

ACCESS GOALS: REASSURANCE · UNDERSTANDING

Accessibility in this project extends beyond technical compliance to account for real-world contexts and barriers. The page is responsive, includes dark mode, uses accessible language, and provides definitions when specialized terms are introduced. Content also directly addresses common concerns for working adults like time commitment, cost, and available support, so users are not left guessing or searching elsewhere for information critical to their decision.

This accessibility extends beyond words. One example is the use of checkmark emojis as answers to the question, "Who Belongs in LXD?" This visual draws attention, to ensure the user is paying attention, then answers each question with an implied, "Of course!" The decision to use symbols rather than text was intentional to avoid patronizing while still providing an affirmative answer.

Revisions & Problem‑Solving

REVISION DRIVER: FEEDBACK · CHANGES: STRUCTURE

Feedback and iteration played a central role in the final design. Peer review revealed that some early enclosure choices increased confusion rather than clarity, leading me to simplify visual grouping and rely more on explicit headers. These revisions were a shift from designing based on intention to designing based on observed impact, prioritizing user understanding over stylistic preference.

Peer review also shined a light on areas where my reassuring tone was taken too far. In one instance, I referred to the technical abilities required as "if you can use a mouse and keyboard, you can succeed in this program." This language was intended to be reassuring but went too far, downplaying the realities of technical expertise needed for success. I revised this to retain the encouraging tone while being more realistic about the skills necessary.

Design Principles in Practice

DESIGN FOCUS: HIERARCHY · COGNITIVE LOAD: REDUCED

Design decisions throughout the page are driven by an intentional focus on how users perceive, process, and prioritize information. Related content is grouped into proximal sections so users can easily understand what belongs together and where to look next. A short-form summary of the page's information is placed to take advantage of preattentive processing, which lessens the burden on users on repeat visits or in a hurry. To avoid overwhelming users, the design also limits how much content is visible at one time, allowing details to be revealed gradually as needed. Together, these choices support clarity, reduce mental effort, and help users engage with the page at their own pace.

Tools & Process

WORKFLOW: ITERATIVE · ROLE: SOLO DESIGNER

The project was built using HTML, CSS, and JavaScript to maintain full control over layout and interaction. Visual Studio Code and Copilot supported rapid iteration and reduced technical friction, while GitHub enabled testing and sharing across devices. These tools supported the design process without becoming the focus of the project itself.

The process of using these technologies was highly educational. My experience with CSS was minimal prior to this project, and it helped me develop that into a competency I am proud of. One example of a learning moment: There is a table that originally used W3Schools CSS code for specific style elements. This code was functional, but it did not take into account the dark mode functionality I built in. It was through peer review that this shortcoming was pointed out and promptly corrected by integrating the table styles into the existing CSS, which allowed it to work in both light and dark mode.

The Final Design

IMPLEMENTATION: COMPLETE · FORMAT: ACCESSIBLE WEB

The final design is a responsive, self contained web experience that presents information about Metro State University’s Learner Experience Design Graduate Certificate in a clear, reassuring, and user centered way. After an orienting page header, a visually emphasized Fast Facts section provides immediate access to high priority information for users who are short on time or revisiting the page. The rest of the content is organized into clearly labeled thematic sections with progressive disclosure used to limit visible information and reduce cognitive load.

Reflection

SKILLS: DESIGN · RHETORIC · ACCESS

This project strengthened my ability to design from a user’s perspective and reinforced that effective design is measured by how well it serves its audience. Iteration and critique revealed the importance of clarity, restraint, and explicit structure when designing for users under cognitive load.

Given more time, I would iterate further on the final section of the page, "Next Steps." This section is one that I struggled to find a balance for. It has undergone changes since the initial version, but I still believe there is room for improvement. I would like to perform user testing and further interviews to make this section more in-line with the overall goal of accessibility and inclusion.
